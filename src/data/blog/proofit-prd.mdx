  <section class="proof-section" id="product">
    <div class="proof-section-heading">
      <p class="proof-label">04 / Product</p>
      <h2>A credibility link for the exact WhatsApp moment.</h2>
    </div>
    <div class="proof-loop">
      <article>
        <span>1</span>
        <h3>Vendor builds profile</h3>
        <p>GST, order range, photos, areas served, languages, and business details live in one public URL.</p>
      </article>
      <article>
        <span>2</span>
        <h3>Buyer asks for details</h3>
        <p>The vendor sends one ProofIt link instead of assembling photos, PDFs, references, and GST screenshots.</p>
      </article>
      <article>
        <span>3</span>
        <h3>Buyer gets confidence</h3>
        <p>The profile answers “is this real?”, “have others used them?”, and “can I forward this internally?”</p>
      </article>
      <article>
        <span>4</span>
        <h3>Vendor captures proof</h3>
        <p>After work is completed, the vendor asks the client for a vouch that updates the profile.</p>
      </article>
    </div>
    <div class="proof-not-box">
      <h3>What ProofIt is not</h3>
      <ul>
        <li>Not a lead marketplace.</li>
        <li>Not a booking platform.</li>
        <li>Not a paid listing badge pretending to be trust.</li>
        <li>Not a website builder. It is proof packaged for procurement.</li>
      </ul>
    </div>
  </section>
